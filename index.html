The 3 Stages of AI Decay

🍯

1. The Honeymoon

Spec → Prompt → PR. It works! The code is sloppy, but who cares? We're flying.

🌫️

2. The Fog

Bugs emerge. You task the AI to fix them. It writes "Circular Tests" that pass its own misconceptions.

🧱

3. The Wall

The codebase exceeds the AI's reliable context window. Adding features is now a "whack-a-mole" nightmare.

Why AI Gets This Wrong

  • RL Bias: Rewarded for immediate "correctness," not long-term elegance.
  • The Median Trap: Trained on the average quality of the web (mostly disposable code).
  • The Missing Loop: AI has never had to maintain a 5-year-old system. It doesn't suffer; you do.
AI Goal "Make it Work"

"Good code lets you change one thing without reading everything."

  • Abstraction as Compression: Interfaces hide complexity from the AI’s context window.
  • Bounded Context: You shouldn't need a map to find the light switch.

The "Primitive Trap" (SKILL Demo)

Stage 1: AI Default (Sloppy)
function processOrder(
  id: string,
  qty: number,
  status: string,
  price: number
) {
  if (status === 'pending') { ... }
  if (status === 'cancelled') { ... }
  // Logic leaks everywhere
}
Stage 2: After Structural Review
type OrderState =
  | PendingOrder
  | ShippedOrder
  | CancelledOrder;

function processOrder(order: OrderState) {
  // Invalid states are impossible.
  // The type handles the logic.
}
Structural Code Analysis Skill: AI identified "Primitive Obsession" and suggested Algebraic Data Types (ADTs).

1

Identify Tensions

2

3

New Definition of Done

  • Self-Documenting: The interface tells the story, not the comments.
  • Durable Structure: Can I change a business rule without touching 10 files?
  • Human Context: "I chose this because..." (not "The AI did it").
