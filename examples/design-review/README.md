# Design Review Skill Example

This is a snapshot of a personal AI review skill I use locally. It is included
as an example of giving an agent a goal, vocabulary, and answer shape.

It is not a standard anywhere, not a complete code-review process, and not a
replacement for human judgment. The two have some overlap, but are somewhat
distinct: quality asks whether the code is clear and honest; structure asks
whether the code shape reduces future context and prevents misuse.

I'm still working on this myself, so take as an example!
